WNMP 环境配置

安装及部署

※PS: (以下文件路径尽量不要带有空格[php可能访问不到MySQL])

!!! 注
    以下各文件解压路径: C:\ini\

!!!文件路径为大写时无法配置成功???

MySQL

mysql-9.3.0-winx64.zip 解压, 复制文件到文件夹(e.g. C:\ini\mysql)

添加环境变量

创建C:\ini\mysql\mysql.ini文件

安装MySQL并完成初始配置

!!! warning
    以上错误需要安装: 微软运行库MSVBCRT.AIO

打开命令行

MySQL命令

Nginx

nginx-1.28.0.zip解压, 复制文件到文件夹(e.g. C:\ini\nginx)

C:\ini\nginx\conf\nginx.conf 配置文件更改

启动: 1) 双击 nginx.exe 启动 2) 命令行执行 start nginx.exe 启动 3)添加到系统服务启动[NSSM]

验证: 浏览器访问 localhost/index.html or 127.0.0.1/index.html, 出现如下页面即访问成功! (默认源文件位于 C:\ini\nginx\html\index.html)

PHP

php-8.4.11-nts-Win32-vs17-x64.zip 解压, 复制文件到文件夹(e.g. C:\ini\php)

复制 C:\ini\php\php.ini-production 文件为 php.ini

php.ini 配置文件更改

启动: 1)命令行执行 php-cgi.exe -b 127.0.0.1:9000 -c php.ini 2)添加到系统服务启动[PS. 利用NSSM设置为服务项时, 若参数的路径中有空格(e.g. C:\Program Files\PHP\php.ini)可能会导致数据库无法使用!!!]

验证: [重启 nginx], 新建 C:\ini\nginx\html\phpinfo.php 文件, 浏览器访问 localhost/phpinfo.php or 127.0.0.1/phpinfo.php, 出现如下页面即配置成功! (前提: nginx.conf 配置文件已修改, 支持 php 解析)

phpMyAdmin

phpMyAdmin-5.2.2-all-languages.zip 解压文件夹重命名为 phpMyAdmin, 复制 文件夹 phpmyadmin 到网站的根目录(e.g. C:\ini\nginx\html\phpmyadmin)

验证: 浏览器访问 localhost/phpmyadmin, 出现如下页面即配置成功!

用户名[默认root]及密码见MySQL配置

※phpMyAdmin Warning及解决措施

! The configuration file needs a valid key for cookie encryption. A temporary key was automatically generated for you. Please refer to the documentation.

–> 复制文件 config.sample.inc.php 为 config.inc.php

! The phpMyAdmin configuration storage is not completely configured, some extended features have been deactivated. Find out why.
Or alternately go to ‘Operations’ tab of any database to set it up there.

–> Create a database named ‘phpmyadmin’ and setup the phpMyAdmin configuration storage there. [创建一个名为’phpmyadmin’的数据库, 并在那里设置phpMyAdmin配置存储。]

WordPress

wordpress-6.8.1-zh_CN.zip 解压, 复制文件到文件夹(e.g. C:\ini\nginx\html)

利用 phpMyAdmin 创建 wordpress 数据库[utf8mb4_general_ci].

浏览器访问 127.0.0.1/index.php 按提示进行安装.

wp-config.php(此文件自动创建)

Enjoy!

### 5)Typecho

typecho.zip 解压, 复制文件到文件夹(e.g. C:\iu\nginx\html\)

利用 phpMyAdmin 创建 typecho 数据库.

浏览器访问 127.0.0.1/install.php 按提示进行安装.

!!! 访问主页 404 Not Found

注册 nginx 和 php-cgi 为系统服务项

NSSM下载

访问 NSSM官网 下载NSSM

此链接可直接下载[最终版] nssm 2.24.zip (2014-08-31)

下载完成后解压得到相关文件, 根据系统选择32位(win32)或 64位(win64)

nginx

将 win64 文件夹下的 nssm.exe 文件复制到 Nginx 根目录下

管理员权限运行 cmd 进到 Nginx 的根目录下, 输入 nssm install nginx

Path: 选择 nginx.exe 文件

Startup directory: nginx.exe文件所在的目录路径[默认选项即可!]

Arguments: 此处参数为空

Service name: nginx

以上选择完成后, 点击 Install service 将 nginx 配置为服务项, 名称 nginx

php-cgi

将 win64 文件夹下的 nssm.exe 复制到 PHP 根目录下

管理员权限运行cmd进到PHP的根目录下, 输入nssm install php

Path: 选择 php-cgi.exe 文件

Startup directory: php-cgi.exe文件所在的目录路径[默认选项即可!]

Arguments: -b 127.0.0.1:9000 -c C:\iu\php\php.ini

以上选择完成后, 点击 Install service 将 php-cgi 配置为服务项, 名称 php

设置开机自启动

Win+R 输入 services.msc 进入服务设置

分别找到 nginx, php 服务项, 将启动方式设置为 自动
